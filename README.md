# MultiLogger

Многопоточная библиотека для записи сообщений в журнал с разными уровнями важности.

## Сборка

```sh
make          # библиотека + приложение + тесты
make test     # собрать и запустить тесты
make clean    # удалить объектные файлы
```

Результат в папке `build/`:
- `libmultilogger.so` — динамическая библиотека
- `logger_app` — консольное приложение
- `logger_test` — тесты

## Использование библиотеки

```cpp
#include "lib/Journal.hpp"

// инициализация: файл журнала и уровень по умолчанию
MultiLogger::FileWriter writer{"app.log", MultiLogger::LogType::Warning};

// запись сообщений
writer.Log("просто сообщение");                    // уровень по умолчанию (Warning)
writer.Log("предупреждение", MultiLogger::LogType::Warning);
writer.Log("ошибка", MultiLogger::LogType::Error);

// смена уровня по умолчанию
writer.DefaultType(MultiLogger::LogType::Message);
```

Сообщения с уровнем ниже заданного в журнал не записываются.
Порядок уровней: `Message` < `Warning` < `Error`.

## Приложение

```sh
./build/logger_app -f <файл> -dt <уровень> [2>> errors.log]
```

- `-f <path>` — путь к файлу журнала
- `-dt <type>` — уровень важности по умолчанию (`Message`, `Warning`, `Error`)
- `2>> errors.log` — перенаправление ошибок в отдельный файл (опционально)

### Интерактивные команды

| Команда | Описание |
|---|---|
| `-h` | справка |
| `-tl` | список уровней важности |
| `-f <path>` | сменить файл журнала |
| `-dt <type>` | сменить уровень по умолчанию |
| `-l [type] <text>` | записать сообщение (тип опционален) |
| `exit` | завершение работы |

### Пример

```sh
./build/logger_app -f ./test.log -dt Error 2>> errors.log

# interactive: запись сообщения с указанием типа
-dt Warning
-l Error что-то пошло не так

# interactive: запись с типом по умолчанию
-l обычное сообщение
```

Формат строки в журнале:

```
[Error][2026-07-17 16:30:00] что-то пошло не так
[Warning][2026-07-17 16:30:05] обычное сообщение
```
